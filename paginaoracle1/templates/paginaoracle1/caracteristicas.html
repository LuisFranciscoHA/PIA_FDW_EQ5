Características

En esta sección conocerás las principales características de Oracle NoSQL Database Cloud Service y cómo sus funciones ayudan a los desarrolladores a trabajar de manera más eficiente y segura.



Características principales

  1. Totalmente gestionado con administración cero
  2. Ciclo de vida de desarrollo más rápido
  3. Alto rendimiento y predecibilidad
  4. Modelado de datos flexible
  5. Seguridad y redundancia
  6. Compatibilidad con datos JSON
  7. Tiempo de duración configurado
  8. Consultas SQL completas

Detalles de cada característica

  • Administración cero: Oracle se encarga del hardware, software, y seguridad para que el desarrollador se enfoque solo en la aplicación.
  • Desarrollo ágil: Permite comenzar a trabajar inmediatamente tras acceder al servicio, sin preocuparse por configuración avanzada.
  • Rendimiento alto: Garantiza respuesta rápida incluso si la carga de trabajo o almacenamiento crece.
  • Modelado híbrido: Soporta datos con y sin esquema, incluyendo objetos JSON.
  • Redundancia de datos: Guarda los datos en múltiples zonas para protegerlos ante fallos.
  • JSON + SQL: Se pueden consultar datos JSON usando SQL, sin complicaciones.
  • Duración limitada: Los datos pueden expirar automáticamente, útil para IoT y sensores.
  • Consultas SQL: Compatible con el lenguaje de consulta más común en bases de datos.

Glosario de términos

Oracle NoSQL Database
Base de datos distribuida que ofrece escalabilidad, rendimiento y disponibilidad para aplicaciones modernas.
Alta disponibilidad
Capacidad del sistema de seguir funcionando incluso si ocurre una falla en parte de la infraestructura.
Redundancia
Mecanismo para guardar copias de seguridad en distintas ubicaciones para proteger los datos.
JSON
Formato ligero para representar datos estructurados, fácil de leer y escribir.
SQL
Lenguaje estructurado de consulta, usado para acceder, modificar y gestionar bases de datos relacionales.
